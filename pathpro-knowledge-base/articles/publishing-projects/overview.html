Publishing Projects

PathPro provides multiple ways to present your project publicly, from sharing the full project interface via a dedicated URL to embedding individual sections as widgets on your own website. This flexibility lets you meet your audience wherever they already are.

Publishing Overview

PathPro uses a hybrid publishing approach: the same project URL serves different views depending on who is viewing it. Admins and team members see the full internal interface — settings, management tools, and internal-only content. Community members and visitors see only the public-facing details: your roadmap, feature voting board, release notes, and other sections you've chosen to make visible.

There are three main strategies for publishing your project:

  • Full Interface (Domain/Subdomain Method) — Publish the entire project via its subdomain URL (e.g., your-project.app.pathpro.co) or a custom domain. Visitors get access to all public sections in one place, with navigation between your roadmap, feature voting, release notes, and more.
  • Full Interface (Embed Method) — Use your project's main embed code to embed the complete project interface directly into any page on your website. This gives visitors the same full experience — roadmap, feature voting, release notes, and all other public sections — without leaving your site. You'll find the embed code in Settings & Design → Widgets.
  • Widgets — Embed individual sections (Feature Voting, Roadmap, Release Notes, Support) anywhere on your own website using iframe snippets. Each widget operates independently, so you can place them on different pages throughout your site.

For a walkthrough of publishing the full interface, setting up direct links, and configuring custom domains, see the Publish Your Project getting started guide and the Custom Domains & Branding article.
